Аутентификация

Авторизация запросов к API зависит от того адресованы они к REST API платформы Эвотор или к API вашего сервиса.

Авторизация запросов к REST API платформы Эвотор

Каждый запрос от вашего сервиса к REST API платформы Эвотор требуется аутентифицировать. Чтобы пройти аутентификацию, запросы должны содержать заголовок x-Authorization. В заголовке необходимо указать токен или API-ключ, который можно получить одним из следующих способов:

  • Если вы реализуете интеграцию c 1c, вы найдёте API-ключ в настройках своего приложения в Личном кабинете клиента.

  • Если вы создаёте iframe-приложение для Личного кабинета, вы можете получить API-ключ из веб-адреса, в GET-параметре token.

    https://partner.org/#/?uuid=01-000000000011111&token=string

  • Также, вы можете получить токен с помощью метода POST, по адресу http://<веб-адрес вашего сервиса>/api/v1/user/token. Этот токен платформа Эвотор присваивает пользователю вашего приложения. Пример POST-запроса:

    curl -X POST --header 'Content-Type: application/json' --header 'Accept: application/json' --header 'Authorization: Bearer или Basic' 'https://partner.com/api/v1/user/token'

Запросы к вашему API отправляет облако Эвотор. Запросы облака Эвотор требуется аутентифицировать.

Аутентификация запросов облака Эвотор к API вашего сервиса

Ваше приложение может отправлять запросы к API вашего сервиса. Например, запросы на создание новых пользователей, изменение номенклатуры в товароучётной системе или, если приложение установлено на терминал, – HTTP-запросы от терминала.

Все запросы к API вашего сервиса проксируются через облако Эвотор. Запросы от облака Эвотор в API вашего сервиса требуется аутентифицировать.

Проксирование запросов через облако Эвотор

Способы аутентификации различаются в зависимости от запросов: * Для аутентифиции запросов на регистрацию (/create) или проверку (/verify) пользователей, а также на включение или выключения тарифа (/tariff), облако Эвотор добавляет в запросы заголовок Authorization с типом Basic или Bearer.

Для Basic-авторизации, вам потребуется предоставить в “Эвотор”" имя пользователя и пароль в формате username:password.

Для Bearer-авторизации вам потребуется предоставить в “Эвотор” токен в формате string.

  • Чтобы аутентифицировать остальные запросы, например, запросы от терминала или запросы на изменение номенклатуры товароучётной системы, облако Эвотор добавляет в запросы заголовок Authorization с типом Bearer. В заголовке указыватеся токен, который ваш сервис присваивает пользователю вашего приложения, после его регистрации (/create) или проверки (/verify).
